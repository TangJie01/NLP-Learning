## 今日学习内容清单

- 基础语法：print、数学运算符、字符串、格式化输出（f-string）
- 变量：赋值、命名规则、多重赋值、类型查看
- 循环：`while`、`for`、`range()`、`break`、`continue`
- 条件判断：`if`、`elif`、`else`、多分支结构
- 函数：`def`、参数（位置/关键字/默认）、`return`、全局/局部变量
- 文件读写：TXT/CSV 读写、追加模式 `'a'`、环境自适应、编码设置
- 类与对象：`class`、`__init__`、实例化、方法调用、默认参数
- 输入交互：`input()`、类型转换、输入验证
- 综合练习：乘客信息录入系统、交互式计算器

## 今日工具链

- Kaggle Notebook（代码运行、数据挂载、Output 面板）
- VS Code（`.ipynb` 运行、终端、Markdown 预览）

## 今日收获

1. **文件读写三模式**：`'r'` 读、`'w'` 覆盖写、`'a'` 追加写，配合环境自适应让代码在 Kaggle 和本地无缝运行。
2. **`class` 是面向对象的核心**：`__init__` 在实例化时自动运行，`self` 指向实例本身，封装后的代码更模块化、易复用。
3. **`input()` 始终返回字符串**：与数字比较必须先 `int()` 或 `float()` 转换，空输入会引发 `ValueError`，需要 `isdigit()` 或 `try/except` 防护。

### 遇到的典型问题

| 问题 | 原因 | 解决方法 |
|---|---|---|
| Kaggle 文件读写报错 | 路径缺少 `/kaggle/working/` | 统一使用 `WORK_DIR` 拼接路径 |
| 图片预览不显示 | VS Code 未打开项目文件夹 | 将整个项目文件夹拖入 VS Code |
| `__init__` 报错 `takes no arguments` | 方法名写成了 `_init_` | 前后各两个下划线 `__init__` |
| 追加内容不换行 | 原文末尾无 `\n` | 追加时写 `\n` 前缀 |
